# Gitの使い方（非エンジニア向け）

※ここでは **git と GitHub の概要** を、非エンジニア向けにやさしくまとめています。

---

## 1. Git と GitHub の違い

- **Git**
  - ファイルやフォルダの「変更履歴」を管理するためのツール
  - 自分のPC（ローカル）で使う
- **GitHub**
  - Gitで管理された履歴を「インターネット上で保存・共有」するサービス
  - チームでの共同作業やバックアップに使う

> まとめると  
> **Git＝履歴管理のしくみ**  
> **GitHub＝その履歴をみんなで共有する場所**

---

## 2. Git を使う目的と基本イメージ

Git は、

- 「いつ」「誰が」「何を」変えたかを記録する
- 過去の状態に戻せる
- チームで編集しても変更を整理できる

ためのツールです。

---

## 3. Git の基本的な使い方

### 3-1. 作業を始める（リポジトリを作成）

プロジェクトのフォルダを「Gitで管理する」状態にする。

```bash
git init
```

- このフォルダの中の変更履歴が記録されるようになる
- これを「リポジトリを作成する」と呼ぶ

---

### 3-2. 既存プロジェクトをコピーする（clone）

GitHub 上のプロジェクトを自分のPCにコピーするときは「クローン」を使う。

```bash
git clone <リポジトリのURL>
```

- GitHub のリポジトリページの「Clone」ボタンからURLをコピーして使う
- クローンすると、自分のPCに「ローカルリポジトリ」が作られる

---

### 3-3. 変更を記録する（add → commit → push）

1. ファイルを編集・追加する
2. 変更をGitに登録（ステージング）する  
   ```bash
   git add <ファイル名>
   # すべての変更をまとめて登録
   git add .
   ```
3. 変更内容を「コミット」として記録する  
   ```bash
   git commit -m "変更内容のメモ"
   ```
4. GitHub にアップロードして共有する  
   ```bash
   git push
   ```

> ローカルで履歴をつけるのが **commit**  
> それをGitHubに送るのが **push**

---

### 3-4. 履歴を確認する

```bash
git log
```

- 過去のコミット（変更履歴）が一覧で表示される
- 誰が・いつ・どんなメッセージで変更したかが分かる

---

### 3-5. 変更を取り消す・戻す（概要だけ）

- 間違って変更した場合は、以下のようなコマンドで元に戻せる

  - `git revert`：特定のコミットを「打ち消す」コミットを作る  
  - `git reset`：もっと前の状態に戻す（少し上級者向け）

> 詳しい使い方は実際に使うタイミングで、都度確認するイメージでOKです。

---

## 4. GitHub の基本的な使い方

### 4-1. リモートリポジトリの作成

1. https://github.com/ にアクセス
2. 新しいリポジトリを作成（New Repository）
3. ここが「オンライン上のプロジェクトの置き場所」になる

---

### 4-2. ローカルとGitHubをつなぐ

ローカルのリポジトリとGitHubのリポジトリを紐づけるには：

```bash
git remote add origin <GitHubのURL>
git push -u origin main
```

- 1回設定しておくと、以後は `git push` だけでOKになる

---

### 4-3. 他の人と共同作業する流れ（ざっくり）

- 他人のリポジトリを自分用にコピー：**fork（フォーク）**
- 自分のフォークで編集・コミット
- 元のリポジトリに変更を提案：**pull request（プルリクエスト）**
- オーナーがレビューし、問題なければ **merge（マージ）** して反映

> 「自分でコピーして作業 → 変更を提案 → OKなら取り込んでもらう」  
> というのが、GitHubでの典型的な共同作業の流れです。

---

### 4-4. 他の人の変更を取り込む

GitHub上の最新状態を自分のPCに反映するときは：

```bash
git pull
```

- リモート（GitHub）の変更を取得し、ローカルに反映する

---

## 5. Git と GitHub を使うメリット

### 5-1. 履歴管理がしやすい

- いつでも「前の状態」に戻せる
- 誰が何を変えたかが明確
- 間違って消しても履歴から復元できる

---

### 5-2. 複数人での共同作業が簡単

- それぞれが自分の環境で作業しつつ、GitHubを通して共有できる
- プルリクエストで、変更内容をレビューしてから取り込める
- 変更が「いつ・どこで」行われたかが追いやすい

---

### 5-3. バックアップと可視化

- GitHubにアップしておけば、PCが壊れてもコードはクラウドに残る
- Web画面で変更履歴やファイルの中身を確認しやすい
- プロジェクトの状態が視覚的に把握しやすくなる
