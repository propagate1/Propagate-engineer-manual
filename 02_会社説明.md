## 1. 企業理念 (Mission)

### 🏆 Mission

私たちは、見込み客、求職者、ファンなど様々な訪問者に「挑戦する企業の価値」を伝え、心を動かし、行動を促すホームページを提供します。

**「挑戦を続け、成果を生むプロ集団」**
エンジニアリングチームは以下の3つを実現し続けます。
1.  **一貫して素晴らしい結果を出せる組織**
2.  **安定したクオリティを維持し続ける組織**
3.  **全メンバーが即戦力として活躍し、同じ基準で成果を出せる組織**

## 2. 事業内容と制作プラン

私たちは「サブスクWEB制作」を主軸に、クライアントのニーズに合わせたWebサイトを提供しています。
エンジニアは、以下のプラン定義に基づいて実装・構築を行います。

### 📋 制作プラン定義
各プランのページ数基準は以下の通りです。この範囲内で構成・実装を行います。

| プラン名 | ページ数基準 | 制作の性質 |
| :--- | :--- | :--- |
| **Basicプラン** | **1ページ** | LP（ランディングページ）形式。コンバージョン特化型。 |
| **Standardプラン** | **2〜6ページ** | 小規模コーポレートサイト。名刺代わりのサイトや採用サイト等。 |
| **Advanceプラン** | **7〜12ページ** | 中規模サイト。情報量が多く、ブランディングやSEOを重視。 |

## 3. 制作フロー (Production Workflow)

案件獲得から納品までの標準的な流れは以下の通りです。
エンジニアは**「2. デモデザイン作成」**から参画し、営業資料をもとに形にしていきます。

1.  **案件獲得 (Sales)**
    * 営業担当がクライアントから案件を受注します。
2.  **デモデザイン作成 (Design & Dev)**
    * **ここからエンジニア担当**
    * 営業から共有される**「ヒアリングシート」**と**「アポ録画」**を確認します。
    * クライアントの要望や雰囲気を汲み取り、デモデザイン（初稿）を作成します。
3.  **初稿提出 (Director Check)**
    * 作成したデモをディレクターが確認し、ディレクターから先方へ提出します。
4.  **フィードバック修正 (Correction)**
    * 先方からのフィードバックをもとに、修正対応を行います。
5.  **サイト完成 (Completion)**
    * 最終確認を経て、納品・公開となります。

## 4. 勤怠・給与ルール (Attendance & Salary)

日々の勤怠管理と、業務委託契約における報酬請求（給与）の手順です。ルールを守り、正確な管理をお願いします。

### 📄 勤怠表（日々の記録）
* **URL**: [勤怠管理シート](https://docs.google.com/spreadsheets/d/1_wderfBz3wDYHK3A9bGTzaFITjE0sYnR1t7Max0KDPs/edit?gid=340726162#gid=340726162)
* **記入ルール**:
    * 他の人の出勤表をコピーして、自分のシートを作成してください。
    * 出勤時間、退勤時間、休憩時間、交通費を自己管理で記入してください。
    * **1日稼働の時は、必ず1時間休憩を取る**ようにしてください。

### 🗓 出勤カレンダー（予定と実績）
* **URL**: [出勤カレンダーシート](https://docs.google.com/spreadsheets/d/1GBTRrQitJEeAzzSWMEImGbBRHh3JLtvl4G0Dx9z7-q4/edit?gid=848786411#gid=848786411)
* **記入記号**:
    * `1` : 1日出勤
    * `0.5` : 半日出勤（途中帰宅、または途中出社）
        * ※シート下部に詳細を記入すること（例：`※渡辺14:00-`）
    * `R` : リモート1日稼働
    * `H` : リモート半日稼働
* **遅刻・欠席時の連絡**:
    * 体調不良や電車遅延などで出社が難しい、または遅刻する場合は、Slackの `[#本社-勤怠]` チャンネルで **@鈴木貴登** と **@菊池駿也** にメンションして連絡してください。

インターン生は「業務委託契約」のため、給与を受け取るには**毎月「請求書」の発行が必要**です。

* **請求時期**: 働いた月の翌月の**月初め**
* **作成方法**:
    * 詳細は[こちらのスライド](https://drive.google.com/file/d/1bES34D29nw97wVsynZQ-SXkygDeVN09C/view?usp=drive_link)を確認してください。
