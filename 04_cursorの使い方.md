# Cursorの使い方（非エンジニア向け）

※ここでは **Cursorの基本的な使い方・考え方・豆知識** をまとめます。  
※初期セットアップ（インストールなど）は「ゼロから」のマニュアルに記載し、ここでは「使い方」に絞ります。

---

## 1. Cursorとは？

- **コード編集用エディタ（VS Code系）＋ AIアシスタントが一体化したアプリ**
- プログラミングのコードを編集しながら、
  - エラーの原因を聞く
  - 修正案を出してもらう
  - 新しいコードを書いてもらう  
  といったことを **同じ画面で完結できる** のが特徴。

イメージ：

- **Next.js**：サイトの「骨組み」（ページ遷移・ルーティングなど）
- **Tailwind CSS**：サイトの「見た目・着せ替え」
- **Cursor**：それらのコードを「AIと一緒に編集するためのアプリ」

---

## 2. よく使うキーボードショートカット（最小セット）

※ ⌘ は Mac の Command キー。Windows の場合は Ctrl と読み替えてOKです。

- **⌘I**  
  → 右側に AI チャットパネルを表示  
  （「このエラー直して」「ここをこう変えたい」と相談）

- **⌘B**  
  → 左側に **ファイル一覧／Git（バージョン管理）** のパネルを表示

- **⌘P**  
  → ファイル名で一瞬ジャンプ  
  （例：「`header.tsx`」と入力してそのファイルを開く）

- **⌘R**  
  → 右側の AI チャットパネルを「新しいチャット」に切り替え

- **⌘Shift+F**  
  → プロジェクト全体検索（全ファイルから文字列を探す）

> うまく動かない場合は、左下の **歯車アイコン → Keyboard** からキーバインドを確認できます。

## 3. フォルダ構成の見方（Next.js + Tailwind の例）

- **`app/`（または `pages/`）**  
  - ページ本体。`app/page.tsx` がトップページ。
- **`components/`**  
  - ボタン・カード・ヘッダーなどの **部品（コンポーネント）置き場**。
- **`public/`**  
  - 画像などの静的ファイル。  
    例：`public/img/hero.jpg` → `/img/hero.jpg` として参照。
- **`styles/globals.css`**  
  - Tailwindを読み込む入口・全体スタイルの調整。
- **`tailwind.config.js`**  
  - カラーやフォントなど「デザインの設定ファイル」。

---

## 4. コンポーネントとは？（超ざっくり）

- Webサイトの「パーツ」の型（ボタン、カード、ヘッダーなど）
- **1か所直すと、同じ部品を使っている全ページに反映される**
  - 例：ボタンコンポーネントの色を変更 → サイト全体のボタンが一括で変更される

---

## 5. Tailwind で見た目を変える（単語を足すだけ）

コードの `className` に「単語」を足していくイメージです。

- 余白：
  - `p-6`（内側の余白）、`m-6`（外側の余白）
- 色：
  - `bg-blue-600`（背景色）、`text-gray-700`（文字色）
- 文字：
  - `text-2xl`, `font-semibold`, `leading-tight`
- 並べ方：
  - `flex`, `grid`, `gap-6`
- レスポンシブ：
  - `md:` を前につける  
    例：`p-4 md:p-8`（スマホでは `p-4`、PCでは `p-8`）

> 迷ったら、**変更したいコードを選択 → 右側のAIパネルにこう聞く**：  
> 「この見出しを少し大きくして、上下の余白も増やしてください」  
> と日本語でOKです。

## 6. 実作業の標準フロー

1. Cursor でプロジェクトフォルダを開く
2. ターミナルを開く（Cursor内でOK）
3. **初回のみ**：  
   ```bash
   npm install
   ```
4. 開発サーバー起動：  
   ```bash
   npm run dev
   ```
5. 表示された URL（例：`http://localhost:3000`）をブラウザで開く
6. `app/page.tsx` や `components/` 内を編集
7. ブラウザが自動リロードされるので見た目を確認
8. 区切りの良いところで Git でコミット → GitHub にプッシュ

---

## 7. `npm run dev` と `npm run build` の違い

- **`npm run dev`（開発モード）**
  - 「下書きプレビュー」
  - 保存のたびに自動更新
  - エラーが画面に赤字で表示
  - **制作中は基本ずっとこれ**

- **`npm run build`（本番チェック）**
  - 「清書＋厳しめチェック」
  - 使っていないコードの整理・最適化
  - 本番で壊れそうなところがあるとエラーで止まる

> `npm run build` で赤いエラーが出たら、  
> エラー文をコピーして Cursor の AI に貼り付け、  
> 「原因と直し方を教えて」と聞けばOKです。

---

## 8. よく使うGitの概念（Cursor上で触ることが多いもの）

- **コミット（commit）**  
  - 変更点をメモ付きで保存  
  - 例：`"ヒーローセクションの見出しを大きく"` など
- **プッシュ（push）**  
  - 保存した履歴を GitHub にアップロード
- **ブランチ（branch）**  
  - 「別冊ノート」のようなもの  
  - `main` を汚さないように、`feat/hero-update` などで作業するのが安全

> コツ：  
> 「色だけ」「余白だけ」など **意味ごとに小さくコミット** すると、後で戻しやすいです。

---

## 9. よくある “詰まり” と対処

- **画面が真っ白**
  - ブラウザの DevTools → Console タブのエラー文をコピー
  - Cursor の AI に貼って「原因と直し方」を聞く

- **画像が表示されない**
  - 画像は `public/` 配下か？
  - パスは `/img/...` など正しいか？
  - 拡張子（.png/.jpg）の大文字小文字ミスがないか？

- **「Port 3000 is in use」と出る**
  - すでに dev サーバーが動いていることが多い
  - 表示された別ポート（例：`http://localhost:3001`）を開く

- **Tailwind が効かない**
  - `globals.css` に `@tailwind base;` `@tailwind components;` `@tailwind utilities;` があるか
  - `tailwind.config.js` の `content` で対象ファイルのパスが合っているか確認

---

## 10. AIへの依頼テンプレ（コピペで使える）

- 「`Hero.tsx` と `Header.tsx` に、このスクリーンショットのデザインを再現してください。画像は `public/img/Hero.png` を使ってください。」
- 「`Hero.tsx` の見出しをもう少し大きくして、右揃えにしてください。」
- 「お問い合わせフォームの送信ボタンの色を青にしてください。」

> ポイント：**「どのファイルの」「どの部分を」「どうしたいか」** を日本語で具体的に書くと、ほぼその通りに返ってきます。

## 11. 用語ミニ辞典

| 用語 | ざっくり意味 |
|------|--------------|
| エディタ（Cursor） | コードを書くアプリ。AIが手伝ってくれる。 |
| Next.js | ページ遷移や表示の「骨組み」フレームワーク。 |
| Tailwind CSS | 見た目を整える「着せ替え用の単語集」。 |
| コンポーネント | サイトの部品（セクション・ボタンなどの再利用パーツ）。 |
| props（プロップ） | コンポーネントに渡す設定（文字・色・サイズなど）。 |
| 開発サーバー | `npm run dev` で動く「下書きプレビュー用サーバー」。 |
| ビルド | `npm run build` で行う「本番用への最終チェック＆変換」。 |
| リポジトリ | プロジェクト一式の保管庫（Git / GitHub）。 |
| ブランチ | 別冊ノートのような作業用ライン。 |
| コミット | 変更をメモ付きで保存すること。 |
| プッシュ | コミットをGitHubなどのリモートに送ること。 |
